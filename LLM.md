## AI Usage Policy
To maintain the security and auditability of the Aegis ecosystem, we have established a strict boundary for the use of large language models within our codebase. The policy is simple: ai may be used exclusively for the development of the frontend client, specifically for the generation of styling, layout components, and purely visual adjustments. We recognize that modern web development is a chaotic landscape of boilerplate, and we are willing to delegate that specific burden to the machines.  
However, the use of ai-generated code is strictly prohibited in any area involving the core backend, the cryptographic modules, or the protocol definitions themselves. Every line of code concerning key exchange, encryption, and the double ratchet implementation must be authored and signed off by a human being. We do not trust probabilistic models to handle deterministic security: we prefer our bugs to be the result of human error we can actually learn from rather than a statistical hallucination. Any pull request found to be using machine-generated logic for backend routing or security-sensitive functions will be rejected: we do not gamble with the integrity of the protocol.
